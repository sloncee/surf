# «Глина» — клиентское приложение гончарной мастерской

Учебный проект летней школы: использование ИИ в промышленной разработке на
примере брифа гончарной мастерской. Задача — клиентское мобильное приложение
для записи на мастер-классы (бэкенд, расписание, кабинеты мастера/владельца
по брифу уже существуют и вне скоупа).

Работа выполнена в три этапа, один за другим — каждый использует артефакты
предыдущего:

| Этап | Папка | Результат |
|---|---|---|
| 1. Аналитика | [`01-analysis/`](01-analysis/README.md) | Требования (BR/FR/NFR), User Stories и Use Cases, архитектура и схема данных, API-контракт, дизайн-бриф |
| 2. Разработка | [`02-development/`](02-development/README.md) | Работающий MVP на React + TS + Vite: 3 фичи целиком (US-002, US-003, US-004), мок-API по контракту |
| 3. Тестирование | [`03-testing/`](03-testing/README.md) | 20 тест-кейсов, 2 найденных и исправленных бага |

Подробности, чек-листы ручной проверки и промпты — в README и `.md`-файлах
каждой папки; здесь — только общая карта проекта.

## Что реализовано (MVP)

- **US-001/US-002 — Расписание и запись на мастер-класс**: список слотов на
  ближайшие занятия, запись с выбором «свои инструменты / прокат», обработка
  нехватки мест/проката.
- **US-003 — Мои записи**: вкладки «Предстоящие / Прошедшие / Отменённые».
- **US-004 — Отмена записи клиентом**: отмена до порога
  `cancellation_deadline_at`, после порога — контакт мастерской.

Остальные User Stories из аналитики (оценка мастера, push-уведомления,
статус «постоянный клиент», авторизация) описаны в требованиях, но не
входили в объём разработки MVP — см. `01-analysis/2-requirements/business-requirements.md`
(приоритизация MoSCoW).

## Как запустить приложение

```bash
cd 02-development/app
npm install
npm run dev
```

Откроется адрес из вывода (обычно `http://localhost:5173`). Бэкенда нет —
данные подставляет мок-адаптер на фикстурах (`02-development/0-decisions/tech-stack.md`).

## Использованные инструменты

Работа над всеми тремя этапами велась с использованием бесплатных версий
Claude, Cursor и Qwen. Конкретные промпты — в `prompts/prompts.md` каждого
этапа.

## Как всё связано

Идентификаторы требований (`FR-xx`, `BR-xx`, `US-xxx`) сквозные — фичи в
`02-development/features/` и баги в `02-development/bugs/` ссылаются на них
напрямую, без пересказа формулировок. Открытые вопросы брифа и принятые для
MVP допущения — в `01-analysis/1-elicitation/customer-questions.md`; на них
опирается часть реализации (например, порог отмены в 2 часа).
