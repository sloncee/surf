# Тест-кейсы: MVP «Гончарная мастерская "Глина"»

Область: три реализованные фичи из `02-development/features/` — US-002
(Запись на мастер-класс, вместе с US-001), US-003 (Мои записи), US-004
(Отмена записи). Формат — по образцу критериев приёмки из
`01-analysis/2-requirements/user-stories.md` (Given/When/Then), сгруппировано
по фиче. Ссылки на требования — те же идентификаторы (FR-xx), что и в
`features/US-00x-....md`, чтобы не дублировать формулировки.

Среда: `npm run dev` в `02-development/app`, мок-адаптер на фикстурах
(`slots.json`, `bookings.json`), см. `02-development/0-decisions/tech-stack.md`.

Статус: ✅ пройден / ❌ не пройден (баг) / — не применимо в MVP (вне скоупа
мока, не баг).

## Сводка

- Тест-кейсов сгенерировано: 20 (TC-01…TC-20).
- Найдено багов: **2** — TC-09 (BUG-001) и TC-18 (BUG-002).
- Оба бага исправлены, регрессия по соседним кейсам (TC-02, TC-15, TC-17)
  проверена вручную повторно после фикса — не сломана.
- Кейсы TC-06, TC-07, TC-20 отмечены как проверенные «по коду» — это
  сознательное ограничение мока (гонки запросов и повторные клики после
  успеха физически не воспроизвести в одиночку в браузере), тот же подход
  уже используется в `features/US-002-zapis-na-master-klass.md`.
