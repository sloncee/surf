# Функциональные требования (ФТ / FR)

Область: клиентское мобильное приложение. Роль пользователя — только **Клиент** (R-028).

## A. Аутентификация и профиль

- **FR-A1.** Пользователь может зарегистрироваться и войти по номеру телефона с
  подтверждением одноразовым кодом (OTP). *(Assumption, см. `customer-questions.md` №6)*
- **FR-A2.** Пользователь может просматривать и редактировать свой профиль (имя,
  телефон, e-mail — опционально).
- **FR-A3.** Профиль отображает признак «постоянный клиент» / статус лояльности,
  если он присутствует в ответе API (логика расчёта — вне скоупа, бэкенд).

## B. Просмотр расписания

- **FR-B1.** Пользователь видит список слотов (занятий) на ближайшие **7 дней** от
  текущего момента по умолчанию (R-027).
- **FR-B2.** Пользователь может изменить период просмотра через фильтр дат (более
  длинный горизонт, чем 7 дней).
- **FR-B3.** Пользователь может фильтровать слоты по программе (новички/круг) и/или
  по мастеру.
- **FR-B4.** Если на выбранный период нет доступных занятий — отображается понятный
  empty state «Пока нет доступных занятий» (R-027).
- **FR-B5.** Карточка слота показывает: дату/время, программу, мастера (с рейтингом),
  длительность, число свободных мест, доступность проката инструментов, цену.
- **FR-B6.** Пользователь может открыть детальную карточку слота с полной информацией
  и описанием программы.

## C. Бронирование

- **FR-C1.** Пользователь может записаться на слот, если в нём есть свободные места.
- **FR-C2.** При записи пользователь указывает, работает своими инструментами/фартуком
  или берёт их напрокат.
- **FR-C3.** Если пользователь выбрал прокат, а свободного прокатного фонда на слот
  не осталось — система не позволяет выбрать эту опцию (или предупреждает и
  предлагает прийти со своими инструментами).
- **FR-C4.** Если к моменту подтверждения брони места закончились (гонка запросов —
  гарантия «0 двойных броней» на стороне бэкенда, R-004), приложение показывает
  понятную ошибку и предлагает выбрать другой слот, не оставляя пользователя в
  неопределённом состоянии.
- **FR-C5.** После успешной записи пользователь видит подтверждение и запись
  появляется в разделе «Мои записи».

## D. Мои записи и отмена

- **FR-D1.** Пользователь видит список своих броней с разделением на предстоящие,
  прошедшие и отменённые.
- **FR-D2.** Пользователь может отменить предстоящую бронь самостоятельно.
- **FR-D3.** Отмена доступна только до наступления порога, который приходит из API
  вместе со слотом/бронью (`cancellation_deadline_at`); MVP-допущение — за 2 часа
  до начала (см. `customer-questions.md` №1).
- **FR-D4.** Если порог отмены прошёл, кнопка отмены недоступна, показывается
  пояснение и контакт мастерской для форс-мажорных случаев.
- **FR-D5.** Если занятие отменяет сама мастерская (форс-мажор), связанная бронь
  автоматически переходит в статус «Отменено мастерской» с указанием причины,
  повторная запись на этот слот невозможна (R-008).

## E. Оценка мастера

- **FR-E1.** После завершения занятия (статус брони «Завершено») пользователь может
  поставить оценку мастеру (шкала 1–5) и оставить необязательный комментарий.
- **FR-E2.** Одна бронь допускает не более одной оценки.
- **FR-E3.** Оценка доступна только для брони в статусе «Завершено» (`completed`) —
  то есть клиент был записан и посетил занятие. Для всех остальных статусов формы
  оценки нет: «Отменено клиентом», «Отменено мастерской», «Неявка» (`no_show`), а
  также для ещё не начавшихся записей. *(Исправлено: ранее статус «Неявка» не был
  явно перечислен, хотя присутствует в модели данных — см. `3-design/data-model.md`
  — и по смыслу не должен давать право на оценку.)*
- **FR-E4.** Пользователь видит агрегированный рейтинг мастера в карточке слота и в
  профиле мастера.

## F. Уведомления

- **FR-F1.** Пользователь получает push-напоминание о предстоящей записи (за
  фиксированное время до начала, значение приходит из конфигурации бэкенда).
- **FR-F2.** Пользователь получает push-уведомление, если мастерская отменила его
  занятие, с указанием причины (R-008).
- **FR-F3.** Пользователь получает push-напоминание оценить мастера после
  завершения занятия.
- **FR-F4.** Пользователь может управлять разрешением на push-уведомления
  (запрос разрешения ОС, деградация функциональности без уведомлений — приложение
  остаётся полностью рабочим, просто без пушей).
