# Бизнес-требования (BR)

| ID | Требование | Источник |
|---|---|---|
| BR-001 | Клиент должен иметь возможность самостоятельно записываться на мастер-класс без участия администратора мастерской | бриф |
| BR-002 | Система должна исключить двойную запись на один слот (гарантия обеспечивается бэкендом, R-004) | бриф, R-004 |
| BR-003 | Система должна ограничивать позднюю отмену брони клиентом, не создавая ощущения давления/недружелюбности к клиенту | бриф |
| BR-004 | Мастерская должна иметь возможность видеть качество работы мастеров через оценки клиентов | бриф |
| BR-005 | Клиент должен вовремя узнавать об отмене занятия мастерской (форс-мажор) | бриф, R-008 |
| BR-006 | Постоянные клиенты должны быть каким-то образом отмечены/поощрены | бриф |
| BR-007 | Онлайн-оплата не является обязательной для запуска MVP | бриф |
| BR-008 | Решение должно быть готово к началу осеннего сезона (ограниченный бюджет и сроки → приоритет базовому функционалу, «навороты» — потом) | бриф |
| BR-009 | Приложение ориентировано в первую очередь на мобильные устройства клиентов | бриф |

## Приоритизация (MoSCoW) для MVP

| Приоритет | Функциональность |
|---|---|
| **Must have** | Просмотр расписания на 7 дней, запись на слот с выбором инструментов/проката, отмена брони с учётом порога, просмотр «Моих записей», отображение отмены мастерской + push, регистрация/вход |
| **Should have** | Оценка мастера после занятия, push-напоминание о предстоящей записи |
| **Could have** | Отображение статуса «постоянный клиент» / бейджей лояльности |
| **Won't have (в этой поставке)** | Онлайн-оплата, кабинет мастера, админка/управление расписанием, SMS/e-mail-уведомления |
