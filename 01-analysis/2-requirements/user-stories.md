# User Stories

Формат: `Как <роль>, я хочу <действие>, чтобы <ценность>` + критерии приёмки в формате
Given/When/Then.

---

## US-001. Просмотр доступных занятий

**Как** клиент, **я хочу** видеть список ближайших мастер-классов с указанием свободных
мест, **чтобы** выбрать удобное время без звонков и переписки в директ.

**Критерии приёмки:**
- Given клиент открыл главный экран, When экран загрузился, Then показан список слотов
  на ближайшие 7 дней, отсортированный по дате/времени.
- Given на ближайшие 7 дней нет ни одного слота, When список загружен, Then показан
  empty state «Пока нет доступных занятий» с возможностью расширить период.
- Given клиент установил фильтр по программе «Круг», When применён фильтр, Then в
  списке остаются только слоты этой программы.

---

## US-002. Запись на мастер-класс

**Как** клиент, **я хочу** записаться на выбранное занятие сам, **чтобы** не ждать
ответа администратора и быть уверенным, что место точно моё.

**Критерии приёмки:**
- Given у слота есть свободные места, When клиент подтверждает запись с выбором
  «свои инструменты», Then создаётся бронь в статусе «Подтверждена», место в слоте
  уменьшается на единицу.
- Given клиент выбрал «прокат», When свободный прокатный фонд слота равен нулю,
  Then опция проката недоступна для выбора (или показано предупреждение).
- Given два клиента одновременно бронируют последнее место, When второй запрос
  приходит после того, как место уже занято, Then второй клиент видит понятное
  сообщение об ошибке и предложение выбрать другой слот (без двойной брони, R-004).

---

## US-003. Просмотр своих записей

**Как** клиент, **я хочу** видеть все свои записи в одном месте, **чтобы** не
держать в голове, когда и куда я записан.

**Критерии приёмки:**
- Given у клиента есть брони, When открыт раздел «Мои записи», Then они разделены
  на вкладки/секции «Предстоящие», «Прошедшие», «Отменённые».
- Given у клиента нет ни одной брони, When открыт раздел, Then показан empty state
  с предложением найти мастер-класс.

---

## US-004. Отмена записи клиентом

**Как** клиент, **я хочу** иметь возможность отменить запись заранее, **чтобы**
освободить место, если планы изменились.

**Критерии приёмки:**
- Given бронь подтверждена и до начала занятия больше порога отмены, When клиент
  нажимает «Отменить», Then бронь переходит в статус «Отменено клиентом», место
  освобождается в слоте.
- Given до начала занятия остаётся меньше порога отмены, When клиент открывает
  карточку брони, Then кнопка отмены недоступна и показано пояснение с контактом
  мастерской.

---

## US-005. Уведомление об отмене занятия мастерской

**Как** клиент, **я хочу** сразу узнать, если мастерская отменила моё занятие
(форс-мажор), **чтобы** не приехать зря.

**Критерии приёмки:**
- Given мастерская отменила слот, When событие обработано, Then связанная бронь
  клиента получает статус «Отменено мастерской» с причиной, клиент получает push.
- Given бронь получила статус «Отменено мастерской», When клиент пытается записаться
  на этот же слот повторно, Then система не позволяет это сделать (слот недоступен
  для записи, R-008).

---

## US-006. Оценка мастера

**Как** клиент, **я хочу** оценить мастера после занятия, **чтобы** поделиться
впечатлением и помочь мастерской видеть качество работы преподавателей.

**Критерии приёмки:**
- Given бронь клиента завершена (занятие состоялось), When клиент открывает карточку
  брони, Then доступна форма оценки (1–5 + комментарий), если оценка ещё не
  поставлена.
- Given оценка уже поставлена для этой брони, When клиент открывает карточку, Then
  показана уже выставленная оценка без возможности создать вторую.
- Given бронь в статусе «Отменено клиентом/мастерской», «Неявка» (`no_show`) или
  ещё не началась, When клиент открывает карточку, Then форма оценки не
  отображается — она доступна только для статуса «Завершено» (FR-E3).

---

## US-007. Напоминание о записи

**Как** клиент, **я хочу** получить напоминание перед занятием, **чтобы** не забыть
и не пропустить оплаченное время мастера.

**Критерии приёмки:**
- Given у клиента есть подтверждённая бронь и push-уведомления разрешены, When
  наступает момент напоминания, Then клиент получает push с датой, временем и
  адресом мастерской.
- Given клиент не дал разрешение на push, When наступает момент напоминания, Then
  push не отправляется, но информация видна в «Моих записях» (NFR-09).

---

## US-008. Отображение статуса постоянного клиента

**Как** постоянный клиент, **я хочу** видеть, что мастерская меня «узнаёт»,
**чтобы** чувствовать, что мне рады, и понимать, какие у меня есть привилегии (если
есть).

**Критерии приёмки:**
- Given бэкенд вернул признак статуса лояльности в профиле, When клиент открывает
  профиль, Then статус отображается (бейдж/подпись).
- Given признак не присутствует в ответе API, When клиент открывает профиль, Then
  блок статуса лояльности скрыт (без ошибок и заглушек).
