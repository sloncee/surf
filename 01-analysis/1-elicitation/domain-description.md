# Описание домена — «Гончарная мастерская "Глина"»

## 1. Контекст

Гончарная мастерская проводит групповые мастер-классы по лепке. Сейчас запись ведётся
вручную (Instagram-директ + бумажный ежедневник владелицы), что приводит к двойным
записям и простоям слотов при поздних отменах.

**Скоуп текущей поставки** (уточнён отдельно, после исходного письма заказчика):
вся серверная инфраструктура (бэкенд, БД, формирование расписания, интерфейсы мастера
и владельца) уже существует. Предмет разработки — **только клиентское мобильное
приложение и API-контракт для него**. Приложение потребляет данные о программах,
мастерах и слотах из существующего бэкенда, но не создаёт и не редактирует их.

## 2. Роли (акторы)

| Роль | Описание | В скоупе текущей поставки |
|---|---|---|
| **Клиент** | Человек, который хочет записаться на мастер-класс | ✅ да — единственная роль поставки (R-028) |
| Мастер | Ведёт занятие | ❌ интерфейс уже существует, вне скоупа |
| Владелец/администратор | Управляет расписанием, мастерами, форс-мажорными отменами | ❌ интерфейс уже существует, вне скоупа |
| Бэкенд (существующая инфраструктура) | Источник истины по слотам, программам, мастерам; гарантирует атомарность бронирования | Внешняя система, интегрируется через API |

## 3. Ключевые сущности домена

- **Программа занятия** — тип мастер-класса: «лепка для новичков» (ручная лепка) или
  «гончарный круг» (для тех, кто уже пробовал). У программы есть уровень, длительность
  (~2–2.5 часа), возможность/необходимость проката инструментов и фартука.
- **Мастер** — сотрудник, ведущий занятие. Имеет накопительный рейтинг на основе оценок
  клиентов.
- **Слот (занятие в расписании)** — конкретное занятие: дата/время, программа, мастер,
  вместимость, число свободных мест, доступный прокатный фонд инструментов/фартуков, цена,
  статус.
- **Бронь** — запись клиента на конкретный слот: выбор «свои инструменты / прокат»,
  статус (подтверждена / отменена клиентом / отменена мастерской / завершена).
- **Оценка мастера** — оценка (и опционально комментарий), которую клиент оставляет
  мастеру после завершённого занятия, привязана к конкретной брони.
- **Клиент** — профиль пользователя приложения; может иметь признак «постоянный клиент»,
  который рассчитывается на бэкенде.

## 5. Границы ответственности (важно для непротиворечивости)

- Создание/редактирование программ, мастеров и слотов расписания — **вне скоупа** (делает
  существующая инфраструктура/админка).
- Атомарность бронирования и её транзакционные детали — **вне скоупа**, закрывается
  API-контрактом (см. `4-api/api-contract.md`).
- Отмена слота целиком (форс-мажор) выполняется в существующей инфраструктуре; клиентское
  приложение только отображает результат (статус брони + уведомление).
- Историческая миграция данных — не требуется, проект учебный (R-015).
