# Этап 1 — Аналитика: «Гончарная мастерская "Глина"»

Клиентское мобильное приложение для записи на мастер-классы + API-контракт для него.
Бэкенд, расписание, кабинеты мастера/владельца — уже существуют, вне скоупа
(см. блок «Уточнение по скоупу» в брифе).

## Структура

| Папка | Содержимое |
|---|---|
| `0-customer-brief/` | Исходный бриф заказчика |
| `1-elicitation/` | Описание домена + уточняющие вопросы и рабочие допущения |
| `2-requirements/` | Бизнес-, функциональные и нефункциональные требования, User Stories, Use Cases |
| `3-design/` | Архитектурный план и схема данных |
| `4-api/` | Требования к API-контракту |
| `5-design-brief/` | Требования к экранам и UX-принципам |
| `prompts/` | Промпты, использованные при генерации этапа |

## Как читать

Рекомендуемый порядок: `1-elicitation` → `2-requirements` → `3-design` → `4-api` →
`5-design-brief`. Все требования и решения ссылаются на идентификаторы бизнес-правил
(BR-xx, R-0xx) из брифа и `domain-description.md`, чтобы не дублировать формулировки
между файлами.

## Ревью

Проведено ревью артефактов на непротиворечивость с брифом и между собой —
результаты и внесённые правки см. в `requirements-review.md`.

## Открытые вопросы

Ряд деталей брифа неоднозначен (точный порог поздней отмены, механика статуса
«постоянный клиент» и др.). Они зафиксированы в
`1-elicitation/customer-questions.md` вместе с рабочими допущениями, принятыми для
MVP, — эти допущения стоит подтвердить с заказчиком до начала разработки.
