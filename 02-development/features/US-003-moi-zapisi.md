# Фича: US-003 — Просмотр своих записей

## Симптом / цель

Клиент должен видеть все свои брони в одном месте — предстоящие, прошедшие
и отменённые — чтобы не держать в голове, когда и куда он записан.

## Требования

- User story: **US-003** (`01-analysis/2-requirements/user-stories.md`)
- Функциональные требования: **FR-D1**
  (`01-analysis/2-requirements/functional-requirements.md`), также FR-C5
  (запись появляется в «Моих записях» после успешной брони).
- Экран: SCR-005 из `5-design-brief/design-brief.md` — упрощено до одного
  экрана с вкладками, без отдельной карточки брони (SCR-006).
- Эндпоинт контракта: `GET /bookings?status=upcoming|past|cancelled`
  (`01-analysis/4-api/api-contract.md`).

## Реализация

Файлы:
- `app/src/screens/MyBookingsScreen.tsx` — вкладки «Предстоящие / Прошедшие /
  Отменённые», список карточек броней, empty state с переходом в расписание.
- `app/src/api/client.ts` — метод `getBookings(status)`.
- `app/src/api/mockAdapter.ts` — обработчик `GET /bookings` с фильтрацией по
  статусу согласно маппингу из контракта.
- `app/src/api/fixtures/bookings.json` — демо-брони для вкладок «Прошедшие»
  и «Отменённые» (и одна «заблокированная» предстоящая для US-004).
- `app/src/App.tsx` — простая навигация «Расписание / Мои записи» через
  `useState` (без роутера).
- `app/src/lib/display.ts` — общие подписи программ и форматирование дат/цен.

Подход: один экран с тремя вкладками. Брони, созданные через «Записаться» в
расписании, попадают во вкладку «Предстоящие» автоматически — общий массив
броней в `mockAdapter.ts`.

Что замокано: `GET /bookings` возвращает брони текущего клиента (`client-me`)
с вложенным объектом `slot` для отображения даты, адреса и цены. Демо-брони
из фикстуры инициализируются при старте мока.

## Промпты

> Реализуй US-003: один экран «Мои записи» с вкладками «Предстоящие / Прошедшие / Отменённые». 
> Брони из US-002 должны автоматически попадать в «Предстоящие» — используй общий массив в mockAdapter.ts. 
> Добавь empty state с кнопкой «Найти занятие» → переход в расписание. 
> Сойздай демо-брони для «Прошедших» и «Отменённых». Навигацию между экранами сделай через useState в App.tsx, без роутера.

## Ручная проверка

- [ ] `npm run dev` → вкладка «Мои записи» в нижней навигации открывается.
- [ ] Вкладка «Предстоящие» → видна демо-бронь на `slot-1` (если не отменена)
      + брони, созданные через «Записаться» в расписании.
- [ ] Вкладка «Прошедшие» → одна завершённая бронь (`booking-demo-past`).
- [ ] Вкладка «Отменённые» → одна бронь «Отменено мастерской» с причиной.
- [ ] Пустая вкладка «Предстоящие» (после отмены всех) → empty state и кнопка
      «Найти занятие» переключает на «Расписание».
