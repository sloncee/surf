# Фича: US-004 — Отмена записи клиентом

## Симптом / цель

Клиент должен иметь возможность отменить предстоящую запись заранее, чтобы
освободить место, если планы изменились. Если порог отмены прошёл — кнопка
недоступна, показывается пояснение и контакт мастерской.

## Требования

- User story: **US-004** (`01-analysis/2-requirements/user-stories.md`)
- Функциональные требования: **FR-D2–FR-D4**
  (`01-analysis/2-requirements/functional-requirements.md`)
- Экран: SCR-006 из `5-design-brief/design-brief.md` — упрощено: отмена
  встроена в карточку брони на экране «Мои записи», без отдельного экрана.
- Эндпоинт контракта: `POST /bookings/{id}/cancel`
  (`01-analysis/4-api/api-contract.md`).
- Порог отмены: `cancellation_deadline_at` приходит из API вместе с бронью
  (MVP-допущение — +2 часа от момента создания для новых броней;
  `customer-questions.md` №1).

## Реализация

Файлы:
- `app/src/screens/MyBookingsScreen.tsx` — кнопка «Отменить запись»,
  подтверждение (BS-003), блокировка по `cancellation_deadline_at` (FR-D4),
  сообщение с телефоном мастерской.
- `app/src/api/client.ts` — метод `cancelBooking(id)`.
- `app/src/api/mockAdapter.ts` — обработчик `POST /bookings/{id}/cancel`:
  проверка порога (422), освобождение места/проката в слоте, статус
  `cancelled_by_client`.
- `app/src/api/fixtures/bookings.json` — `booking-demo-locked`: предстоящая
  бронь с прошедшим `cancellation_deadline_at` для проверки FR-D4 без ожидания.

Подход: клиент сверяет `cancellation_deadline_at` локально и не показывает
кнопку отмены, если порог прошёл (FR-D4). Серверная проверка в моке — запасная
защита (422). Перед отменой — короткое подтверждение «Место освободится…».

Что замокано: при успешной отмене `free_seats` (+ `free_rental_kits`, если
бронь была с прокатом) возвращаются в слот. Отменённая бронь переезжает во
вкладку «Отменённые».

## Промпты

> Реализуй US-004: кнопка «Отменить запись» прямо в карточке брони на экране US-003, 
> отдельный экран не нужен. Перед отменой — подтверждение. Если cancellation_deadline_at прошёл 
> — кнопка заблокирована, показывается текст с телефоном мастерской (FR-D4). 
> При отмене возвращай free_seats/free_rental_kits в слот, меняй статус на cancelled_by_client, 
> переноси бронь во вкладку «Отменённые». Добавь в фикстуру booking-demo-locked с прошедшим дедлайном.

## Ручная проверка

- [ ] Записаться на занятие → «Мои записи» → «Предстоящие» → «Отменить запись»
      → подтвердить → бронь исчезает из «Предстоящих», появляется в «Отменённых»,
      счётчик мест в расписании увеличивается на 1.
- [ ] Демо-бронь `booking-demo-locked` → кнопки отмены нет, показан текст с
      телефоном мастерской — FR-D4.
- [ ] Вкладка «Отменённые» → бронь «Отменено мастерской» показывает причину
      (форс-мажор, BS-004).
