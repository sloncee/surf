# Этап 2 — Разработка: «Гончарная мастерская "Глина"»

Статус: реализованы **три фичи целиком** — US-002 «Запись на мастер-класс»
(вместе с US-001 «Просмотр доступных занятий»), US-003 «Просмотр своих записей»,
US-004 «Отмена записи клиентом». Каркас простой: два экрана, переключение через
`useState`, мок-API на фикстурах. На этапе тестирования (`03-testing/`) найдены
и исправлены 2 бага — см. `bugs/BUG-001-...md`, `bugs/BUG-002-...md`.

## Важное допущение

По брифу бэкенд уже существует (см.
`01-analysis/1-elicitation/domain-description.md`). По факту для учебной
задачи его нет, поэтому вместо HTTP-вызовов используется **мок-адаптер**
(`app/src/api/mockAdapter.ts`) — он подставляет данные из фикстур вместо
реальных ответов сервера, но по формату путей/ошибок соответствует
`01-analysis/4-api/api-contract.md`. Экраны и `client.ts` написаны так, будто
бэкенд настоящий — при его появлении меняется только `mockAdapter.ts`.

## Как запустить

```bash
cd app
npm install
npm run dev
```

Откроется адрес из вывода (обычно `http://localhost:5173`). В браузере — два
экрана с нижней навигацией «Расписание / Мои записи» (без роутера, см.
`0-decisions/tech-stack.md`).

## Как проверить фичи

Полные чек-листы — в `features/US-00x-....md`, раздел «Ручная проверка».

**US-002 — Запись (экран «Расписание»):**
1. Видно 3 предстоящих занятия с датой, местами, прокатом и ценой.
2. «Записаться» с выбором «Свои / Прокат» → «Вы записаны», счётчик мест уменьшается.
3. Слот без мест — кнопка неактивна; без проката — ошибка при выборе проката.

**US-003 — Мои записи:**
1. Вкладки «Предстоящие / Прошедшие / Отменённые» с карточками броней.
2. Новая бронь из расписания появляется во «Предстоящих».
3. Пустой список → empty state, кнопка «Найти занятие» ведёт в расписание.

**US-004 — Отмена:**
1. «Отменить запись» → подтверждение → бронь уходит в «Отменённые», место
   возвращается в расписание.
2. Демо-бронь с прошедшим порогом отмены — кнопки нет, показан телефон мастерской.

Обновление страницы сбрасывает брони, созданные в сессии (кроме демо-фикстур,
которые инициализируются заново) — ожидаемо для мока в памяти.
